# Security Policy

## Supported Versions

ReactPyx is currently in **alpha** and not recommended for production use.
Security reports are still welcome and appreciated.

## Reporting a Vulnerability

If you discover a security vulnerability in ReactPyx:

1. **Do not open a public GitHub issue with details.**
2. Instead, please contact the maintainers privately (for example through
   a private GitHub security advisory or direct message).
3. Include:
   - A description of the vulnerability.
   - Steps to reproduce, if possible.
   - Any potential impact you have identified.

We will:

- Acknowledge receipt as soon as reasonably possible.
- Investigate the issue.
- Work on a fix or mitigation.
- Coordinate a disclosure timeline if necessary.
